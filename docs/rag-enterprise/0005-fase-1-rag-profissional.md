# 0005 — Fase 1: RAG Profissional

## Objetivo da Fase

Fortalecer o sistema para operação profissional e preparar a transição do backend técnico para um produto com primeira camada visual operacional.

---

## Resultado Esperado da Fase

Ao fim da Fase 1, o programa deve ser:

- tecnicamente confiável
- observável
- mais fácil de operar
- melhor para validação de qualidade
- pronto para receber um frontend operacional simples

---

## Escopo da Fase 1

### 1. Retrieval mais forte

- reranking quando evidência justificar
- filtros melhores
- explicabilidade melhor de retrieval
- tuning mais controlado

### 2. Observabilidade operacional

- métricas mais claras
- groundedness melhor reportado
- dashboards mínimos operacionais
- trilha de auditoria de queries e ingestão

### 3. Avaliação mais robusta

- dataset maior
- execução recorrente de avaliação
- triagem de qualidade
- revisão humana em casos frágeis

### 4. Primeira interface web operacional

Esta fase já pode incluir frontend técnico simples:

- página de upload
- lista de documentos
- status de ingestão
- tela de busca técnica
- tela de perguntas
- visualização de chunks
- visualização de scores
- página simples de métricas

---

## O que entra

- qualidade de retrieval
- qualidade de operação
- observabilidade
- primeira camada visual

## O que não entra

- frontend comercial final
- multitenancy enterprise
- white-label completo
- billing
- painel executivo completo

---

## Gate da Fase 1

A Fase 1 é considerada concluída quando houver:

- qualidade prática melhor que a Fase 0
- observabilidade operacional suficiente
- painel técnico simples funcional
- base estável para produto premium
